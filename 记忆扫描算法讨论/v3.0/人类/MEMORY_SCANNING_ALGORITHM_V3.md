# 记忆扫描算法 3.0 - 渐进式全量检索

**版本**: 3.0  
**创建时间**: 2026-04-02  
**算法类型**: 渐进式全量记忆检索  
**核心原则**: 结构的建立基于客观情况，而非主观判断

---

## 一、算法核心原则

### 1.1 基本原则

**客观性原则**：
- ✅ 结构的建立基于客观情况
- ❌ 不基于主观判断或主观分类
- ✅ 指针、索引自然涌现
- ✅ 停止条件基于客观信号

**不需要向量嵌入**：
- 使用暴力文本比对
- 两两比对保证不超出上下文
- 简单高效，无需额外依赖

**渐进式检索**：
- 从精确匹配开始
- 逐渐放宽搜索条件
- 最终退化为全量枚举

**多跳索引**：
- 问题随着线索演化
- 每次跳跃创建指针
- 形成自然的路由网络

## 二、算法流程

### 2.1 初始化阶段

```
输入: 初始问题 Q0
输出: 相关记忆集合 R, 指针集合 P

1. 设置当前问题 Q = Q0
2. 设置搜索起点 = 最新记忆位置
3. 初始化结果集 R = []
4. 初始化指针集 P = []
5. 初始化问题演化历史 H = [Q0]
```

### 2.2 主检索循环

```
while (未满足停止条件) {
  
  // 1. 从起点向前扫描
  for (当前记忆 M from 起点向前) {
    
    // 2. 两两比对
    相似度 = 比对(Q, M.content)
    
    // 3. 判断是否相关
    if (相似度 >= 当前阈值) {
      
      // 4. 添加到结果集
      R.add(M)
      
      // 5. 创建指针（在最后一条记忆位置）
      指针 = {
        source: 最新记忆ID,
        target: M.id,
        question: Q,  // 当前问题
        summary: 生成摘要(M.content),  // 简短描述
        relevance: 相似度
      }
      P.add(指针)
      
      // 6. 检查是否需要演化问题
      if (发现新线索(M)) {
        新问题 = 演化问题(Q, M)
        Q = 新问题
        H.add(新问题)
        
        // 7. 重置搜索起点（从当前位置继续）
        break  // 跳出内层循环，继续外层循环
      }
    }
    
    // 8. 检查指针跳转
    指针匹配 = 检查指针(M, Q)
    if (指针匹配) {
      目标记忆 = 指针跳转(指针匹配)
      R.add(目标记忆)
      // 指针加速检索
    }
  }
  
  // 9. 渐进式放宽条件
  当前阈值 = 降低阈值(当前阈值)
  
  // 10. 检查是否需要全量搜索
  if (当前阈值 <= 最小阈值) {
    执行全量枚举()
    break
  }
}
```

### 2.3 问题演化机制

```
function 演化问题(当前问题 Q, 新线索 M) {
  
  // 从记忆中提取新线索
  新关键词 = 提取关键词(M.content)
  
  // 更新问题
  新问题 = {
    原始问题: Q,
    新增线索: 新关键词,
    组合问题: Q + " AND " + 新关键词
  }
  
  return 新问题
}
```

## 三、指针机制详解

### 3.1 指针创建规则

**何时创建**：
- 找到相关记忆时（相似度 >= 阈值）
- 问题演化时（发现新线索）
- 指针跳转成功时

**指针位置**：
- **总是创建在最新记忆位置**
- 指向找到的相关记忆
- 记录当时的问题和摘要

**指针数据结构**：
```typescript
interface Pointer {
  id: number;
  sourceId: number;        // 最新记忆ID（指针所在位置）
  targetId: number;        // 指向的记忆ID
  question: string;        // 创建时的问题
  questionKeywords: string[];  // 问题关键词
  summary: string;         // 目标记忆的简短描述
  relevance: number;       // 相关性分数
  hopCount: number;        // 第几跳（0=初始问题，1=第一次演化...）
  createdAt: Date;
  accessCount: number;     // 被使用次数
  lastAccessed: Date;
}
```

### 3.2 多跳索引示例

```
初始问题: "项目进度"
  ↓
找到记忆 M1: "上周完成了数据库设计"
  ↓ 创建指针 P1（在最新记忆位置）
  ↓ 问题演化为: "项目进度 AND 数据库设计"
  ↓
找到记忆 M2: "数据库设计文档在 /docs/db.md"
  ↓ 创建指针 P2（在最新记忆位置）
  ↓ 问题演化为: "项目进度 AND 数据库设计 AND 文档位置"
  ↓
找到记忆 M3: "文档由张三负责维护"
  ↓ 创建指针 P3（在最新记忆位置）

最终结果:
- R = [M1, M2, M3]
- P = [P1, P2, P3]  // 三个指针都在最新记忆位置
- H = ["项目进度", "项目进度 AND 数据库设计", "项目进度 AND 数据库设计 AND 文档位置"]
```

下次查询"项目进度"时：
- 直接使用 P1 跳转到 M1
- 使用 P2 跳转到 M2
- 使用 P3 跳转到 M3
- **无需重新扫描**

## 四、渐进式搜索策略

### 4.1 阈值递减策略

```
初始阈值: 0.9  （高精度匹配）
  ↓
第1轮: 0.9  （精确匹配）
第2轮: 0.7  （高度相关）
第3轮: 0.5  （中等相关）
第4轮: 0.3  （弱相关）
第5轮: 0.0  （全量枚举）
```

### 4.2 问题放宽策略

```
初始问题: "数据库设计文档在哪里？"
  ↓
第1轮: 精确匹配 "数据库设计文档"
  ↓
第2轮: 放宽为 "数据库" OR "文档"
  ↓
第3轮: 放宽为 "数据" OR "文档"
  ↓
第4轮: 放宽为 任意关键词
  ↓
第5轮: 全量扫描
```

## 五、停止机制设计

### 5.1 自适应停止条件

**设计原则**：
- ✅ 基于客观指标，非主观设定
- ✅ 自适应调整，非固定阈值
- ✅ 自组织演化，非外部控制

### 5.2 停止信号组合

#### 信号1: 信息增益衰减

```
信息增益 = 新发现记忆的信息量 / 已检索记忆总数

停止条件: 连续 N 次迭代的信息增益 < 阈值

客观依据: 信息论 - 边际效用递减
```

**实现**：
```typescript
function 计算信息增益(新记忆, 已有记忆集) {
  // 新记忆与已有记忆的差异度
  差异度 = 1 - max(相似度(新记忆, 已有记忆集))
  
  // 信息增益
  增益 = 差异度 / 已有记忆集.length
  
  return 增益
}

// 自适应阈值
阈值 = 历史平均增益 * 衰减系数
```

#### 信号2: 问题收敛

```
问题演化速度 = 新问题与旧问题的差异度

停止条件: 问题不再演化（连续 N 次问题相同）

客观依据: 问题空间已探索完毕
```

**实现**：
```typescript
function 检查问题收敛(问题历史) {
  if (问题历史.length < 3) return false
  
  最近问题 = 问题历史.slice(-3)
  
  // 计算问题相似度
  相似度 = [
    相似度(最近问题[0], 最近问题[1]),
    相似度(最近问题[1], 最近问题[2])
  ]
  
  // 如果问题不再变化
  return 相似度[0] > 0.95 && 相似度[1] > 0.95
}
```

#### 信号3: 记忆饱和

```
工作记忆容量 = 7 ± 2 （Miller's Law）

停止条件: 找到的记忆数量达到工作记忆容量

客观依据: 人类认知限制
```

**实现**：
```typescript
const 工作记忆容量 = 7  // 可根据用户调整

function 检查记忆饱和(结果集) {
  return 结果集.length >= 工作记忆容量
}
```

#### 信号4: 相关性衰减

```
距离衰减 = 距离最新记忆的时间跨度

停止条件: 连续 M 条记忆都不相关

客观依据: 时间距离越远，相关性越低
```

**实现**：
```typescript
function 检查相关性衰减(连续不相关计数, 最大容忍度) {
  return 连续不相关计数 >= 最大容忍度
}

// 自适应容忍度
最大容忍度 = min(100, 总记忆数 * 0.1)  // 最多容忍10%的记忆
```

#### 信号5: 资源预算

```
计算预算 = 可用时间 / 平均检索速度

停止条件: 达到计算预算

客观依据: 物理资源限制
```

**实现**：
```typescript
function 检查资源预算(开始时间, 最大时间) {
  已用时间 = now() - 开始时间
  return 已用时间 >= 最大时间
}

// 自适应时间预算
最大时间 = min(5000ms, 用户平均响应时间 * 0.5)
```

### 5.3 组合停止策略

```typescript
function 检查停止条件(状态) {
  const 信号 = {
    信息增益衰减: 检查信息增益衰减(状态),
    问题收敛: 检查问题收敛(状态.问题历史),
    记忆饱和: 检查记忆饱和(状态.结果集),
    相关性衰减: 检查相关性衰减(状态.连续不相关),
    资源预算: 检查资源预算(状态.开始时间, 状态.最大时间)
  }
  
  // 自适应权重（根据历史数据学习）
  const 权重 = {
    信息增益衰减: 0.3,
    问题收敛: 0.2,
    记忆饱和: 0.2,
    相关性衰减: 0.2,
    资源预算: 0.1
  }
  
  // 加权求和
  const 停止分数 = 
    信号.信息增益衰减 * 权重.信息增益衰减 +
    信号.问题收敛 * 权重.问题收敛 +
    信号.记忆饱和 * 权重.记忆饱和 +
    信号.相关性衰减 * 权重.相关性衰减 +
    信号.资源预算 * 权重.资源预算
  
  // 自适应阈值（根据用户反馈学习）
  const 停止阈值 = 0.6  // 初始值，后续通过学习调整
  
  return 停止分数 >= 停止阈值
}
```

### 5.4 后天学习机制

**用户反馈学习**：
```typescript
interface Feedback {
  查询: string;
  结果集: Memory[];
  用户满意度: number;  // 1-5星评分
  是否采纳: boolean;   // 用户是否使用了结果
  停止信号: StopSignals;
}

function 学习停止阈值(反馈历史) {
  // 分析高满意度查询的停止信号
  高满意度查询 = 反馈历史.filter(f => f.用户满意度 >= 4)
  
  // 统计停止信号分布
  信号分布 = 统计(高满意度查询.map(f => f.停止信号))
  
  // 更新权重
  新权重 = 优化权重(信号分布)
  
  // 更新阈值
  新阈值 = 优化阈值(高满意度查询)
  
  return { 新权重, 新阈值 }
}
```

**自适应调整**：
```typescript
function 自适应调整(用户反馈) {
  if (用户反馈.结果太少) {
    // 降低停止阈值，允许更多搜索
    停止阈值 *= 0.9
    最大容忍度 *= 1.1
  }
  
  if (用户反馈.结果太多) {
    // 提高停止阈值，提前结束
    停止阈值 *= 1.1
    最大容忍度 *= 0.9
  }
  
  if (用户反馈.响应太慢) {
    // 减少计算预算
    最大时间 *= 0.8
  }
  
  if (用户反馈.找到了想要的内容) {
    // 强化当前停止策略
    强化权重(当前停止信号)
  }
}
```

## 六、路径依赖与探索机制

### 6.1 路径依赖原理

**大自然的答案**：
> 如果某个线索（思路）上次有用，那么下次就倾向于使用。

**统计学本质**：

路径依赖是**统计学结果**，而非主观偏好：

```
路径依赖 = 利用已知解决未知
         = 使用最大概率去检索
         = 先检索高概率事件，再检索低概率事件
```

**关键区分**：

| 概念 | 说明 |
|------|------|
| **问题发生的概率** | 某类问题出现的频率 |
| **解决问题的概率** | 某个线索能解决问题的可能性 |

**我们关注的是"解决问题的概率"**，而不仅仅是"问题发生的概率"。

**冷门问题的例子**：

```
场景：遇到一个冷门问题

问题发生的概率：低（很少出现）
解决问题的概率：高（仅有的线索就是关键）

对于"解决这个问题"所需的线索中：
- 虽然线索数量少（冷门）
- 但这些线索是解决问题的最大概率事件
- 因此应该优先使用这些线索
```

**基本原理**：
- 根据使用频率选择路线
- 使用次数最多的路线 = 最短路线
- 这是自然界的基本优化策略

**在记忆系统中的体现**：
```
指针访问次数 → 路径权重
高访问指针 → 优先跳转（解决问题的概率高）
低访问指针 → 备选路径（解决问题的概率低）
```

**检索顺序**：
```
1. 先检索高概率事件（高频指针）
2. 再检索低概率事件（低频指针）
3. 最后全量枚举（兜底策略）
```

### 6.2 路径依赖的问题

**问题**：当问题无解时，如何进行全量搜索而不是路径依赖？

**现象**：
- 系统总是沿着熟悉的路径搜索
- 忽略了可能存在的新路径
- 导致某些问题永远找不到答案

### 6.3 打破路径依赖的机制

**核心洞察**：
> 这属于更高层设计的问题，不是记忆系统本身需要设计的问题。

**机制设计**：

```
用户反馈：不行
    ↓
高层指令：更好地服务用户
    ↓
下次搜索：排除当前线索
    ↓
线索选择：没有产生路径依赖的线索
    ↓
结果：新的路径探索
```

**实现方式**：

```typescript
interface SearchContext {
  当前问题: string;
  已排除线索: string[];  // 用户反馈不行的线索
  路径历史: Path[];
}

function 选择线索(问题: string, 上下文: SearchContext): 线索[] {
  // 获取所有候选线索
  const 候选线索 = 获取所有线索(问题);
  
  // 排除已失败的线索
  const 可用线索 = 候选线索.filter(线索 => 
    !上下文.已排除线索.includes(线索)
  );
  
  // 按路径依赖排序（使用频率）
  const 排序线索 = 可用线索.sort((a, b) => 
    获取使用频率(b) - 获取使用频率(a)
  );
  
  return 排序线索;
}
```

### 6.4 层级划分

**记忆系统层**（本算法）：
- 提供路径依赖机制
- 记录线索使用频率
- 支持线索排除

**高层决策层**（核心指令系统）：
- 接收用户反馈
- 决定是否排除线索
- 触发新的路径探索

**交互示例**：

```
第1次搜索：
  问题: "项目进度"
  路径: 项目进度 → 数据库设计 → 文档位置
  结果: 找到文档位置
  
用户反馈: 不是这个文档

第2次搜索：
  问题: "项目进度"
  排除线索: ["数据库设计", "文档位置"]  // 高层决策
  新路径: 项目进度 → 前端开发 → 部署计划
  结果: 找到正确的进度信息
```

### 6.5 设计原则

**客观性**：
- ✅ 路径依赖基于使用频率（可统计）
- ✅ 线索排除基于用户反馈（可记录）
- ✅ 新路径探索基于排除后的候选集（可计算）
- ❌ 不依赖主观判断

**自组织性**：
- 高频路径自然浮现
- 失败路径自然淘汰
- 新路径自然探索

## 七、算法伪代码

```typescript
async function 记忆检索3_0(初始问题: string): Promise<检索结果> {
  
  // 初始化
  let 当前问题 = 初始问题
  let 当前阈值 = 0.9
  let 结果集 = []
  let 指针集 = []
  let 问题历史 = [初始问题]
  let 连续不相关 = 0
  let 开始时间 = Date.now()
  
  // 主循环
  while (!检查停止条件({结果集, 问题历史, 连续不相关, 开始时间})) {
    
    // 从最新记忆向前扫描
    let 当前记忆ID = await 获取最新记忆ID()
    
    while (当前记忆ID !== null) {
      
      // 两两比对
      const 记忆 = await 获取记忆(当前记忆ID)
      const 相似度 = 比对文本(当前问题, 记忆.content)
      
      if (相似度 >= 当前阈值) {
        // 找到相关记忆
        结果集.add(记忆)
        连续不相关 = 0
        
        // 创建指针
        const 指针 = {
          sourceId: await 获取最新记忆ID(),
          targetId: 记忆.id,
          question: 当前问题,
          summary: 生成摘要(记忆.content),
          relevance: 相似度,
          hopCount: 问题历史.length - 1
        }
        指针集.add(指针)
        await 保存指针(指针)
        
        // 检查是否需要演化问题
        if (发现新线索(记忆)) {
          当前问题 = 演化问题(当前问题, 记忆)
          问题历史.add(当前问题)
          break  // 重新开始扫描
        }
      } else {
        连续不相关++
      }
      
      // 检查指针跳转
      const 匹配指针 = await 检查指针(当前记忆ID, 当前问题)
      if (匹配指针) {
        const 目标记忆 = await 获取记忆(匹配指针.targetId)
        结果集.add(目标记忆)
        await 更新指针访问(匹配指针)
      }
      
      // 移动到前一条记忆
      当前记忆ID = await 获取前一条记忆ID(当前记忆ID)
    }
    
    // 渐进式放宽条件
    当前阈值 -= 0.2
    
    if (当前阈值 < 0.0) {
      // 全量枚举
      break
    }
  }
  
  return {
    结果: 结果集,
    指针: 指针集,
    问题演化: 问题历史,
    统计: {
      扫描记忆数: 统计扫描数(),
      找到记忆数: 结果集.length,
      问题演化次数: 问题历史.length - 1,
      用时: Date.now() - 开始时间
    }
  }
}
```
