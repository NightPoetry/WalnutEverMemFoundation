## 版本演进

### v1.0 - 基于文件的记忆扫描算法
- **存储**: Markdown 文件（`memory/YYYY-MM-DD.md`）
- **匹配**: 文本关键词匹配
- **索引**: 扫描结果记录在文件中
- **检索**: 线性扫描历史文件
- **核心原则**: 如无必要，勿建索引

### v2.0 - 基于数据库的记忆扫描算法
- **存储**: SQLite/PostgreSQL（支持向量）
- **匹配**: 向量嵌入相似度（RAG 技术）
- **索引**: 指针数据结构（带元数据）
- **检索**: O(1) 指针跳转 + 向量搜索
- **核心原则**: 继承 v1.0 思路，优化实现方式

### v3.0 - 渐进式全搜索算法（设计ing）
- **存储**: SQLite（无需向量嵌入）
- **匹配**: 问题演化 + 两两比对
- **索引**: 多跳指针（在最新记忆位置创建）
- **检索**: 渐进式扩展 + 自适应停止
- **核心原则**: 结构的建立基于客观情况，而非主观判断

## 版本管理规则

### 版本命名
- **v1.0** - 基于文件的原始实现
- **v2.0** - 数据库+向量优化版本
- **v3.0** - 渐进式全搜索（无向量）

### 文档分类
- **人类/** - 人类设计师编写的文档
- **AI/** - AI 助手生成的思考和建议

### 文档命名规范
- 使用清晰的英文名称
- 使用下划线分隔单词
- 使用 .md 扩展名
- 示例：`MEMORY_SCANNING_ALGORITHM_V3.md`

## 讨论流程

1. **人类提出设计** → 在 `人类/` 目录创建文档
2. **AI 分析思考** → 在 `AI/` 目录创建响应文档
3. **人类反馈调整** → 更新或创建新版本
4. **迭代优化** → 形成最终方案
